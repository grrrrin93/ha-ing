1. PRD (제품 요구사항 정의서)
1.1 문제 정의 (Problem Statement)
"영어 일기를 써도 이게 원어민이 진짜 쓰는 말인지 모르겠고, 문법만 고쳐주는 기존 앱은 너무 지루해요." 30대 직장인들은 출퇴근 시간이라는 짧은 틈을 타 '진짜 쓸모 있는' 영어를 배우고 싶어 하지만, 자신의 의도가 담긴 긴 문장을 세련되게 다듬어주는 서비스의 부재로 인해 학습 효능감을 느끼지 못하고 있습니다.

1.2 제품 목표 (Goals)
의도 중심 교정: 사용자의 원래 뜻은 보존하면서 표현만 '원어민 문어체'로 업그레이드.

실전 활용 연결: 일기 내용을 바탕으로 즉시 스몰토크에 활용할 수 있는 '구어체' 제공.

습관 형성: 듀오링고와 같은 활기찬 UX를 통해 출퇴근길 10분 학습을 즐겁게 만듦.

1.3 사용자 페르소나
이름: 김프로 (31세, 마케터)

상황: 지하철로 왕복 1시간 반 출퇴근. 외국인 파트너사와 협업이 잦아 세련된 영어 표현에 목마름.

니즈: 내가 오늘 겪은 일을 영어로 멋지게 설명하고 싶음. 딱딱한 교과서 영어 탈피.

1.4 사용자 스토리
작성: 사용자는 스마트폰으로 오늘의 일상을 자유롭게 영작할 수 있어야 한다.

교정: 사용자는 내 글이 '세련된 문어체'와 '친근한 구어체' 두 버전으로 변환된 것을 확인할 수 있어야 한다.

확장: 사용자는 주요 표현(이디엄)이 포함된 추가 대화 예시를 통해 학습 범위를 넓힐 수 있어야 한다.

1.5 성공 지표 (Success Metrics)
실전 활용률: 학습한 표현의 50% 이상을 실제 대화에서 활용했다고 사용자가 체크.

유지율(Retention): 일주일 중 5일 이상 방문하여 일기를 작성하는 사용자의 비율.

1.6 범위 외 항목 (Non-goals)
실시간 화상 영어 연결 (현재 단계에서는 AI로만 대체)

전문적인 시험(TOEIC, TOEFL) 대비 문제 풀이 기능

2. TRD (기술 요구사항 정의서)
2.1 시스템 아키텍처
사용자(Web) <-> Vercel(Hosting) <-> Next.js(Frontend) <-> Supabase(DB/Auth) <-> Gemini Pro API(AI)

2.2 기술 스택
프론트엔드: Next.js (React) - 반응형 웹 구현 및 빠른 초기 로딩에 최적.

백엔드/DB: Supabase - 서버리스 환경으로 1인 운영 관리 포인트 최소화. 무료 티어로 충분한 텍스트 저장 가능.

AI 모델: Google Gemini 1.5 Pro API - 무료 티어를 활용한 고성능 텍스트 교정 및 대화 생성.

배포: Vercel - CI/CD 자동화로 코드 수정 시 즉시 서비스 반영.

2.3 데이터 요구사항
개인정보 최소화: 소셜 로그인(Google)을 통해 필수 정보만 수집.

보존 기간: 사용자가 탈퇴하기 전까지 클라우드에 영구 보존(성장 추적용).

6. TASKS (AI 개발 파트너용 프롬프트 설계서)
[M0] 프로젝트 초기화
컨텍스트: Next.js와 Supabase를 연결한 기본 뼈대 구축.

태스크: npx create-next-app으로 프로젝트를 생성하고, Supabase Auth(Google)를 연동하세요. 폴더 구조는 /src/app 기반의 App Router를 사용합니다.

[M1] 디자인 시스템 및 레이아웃
컨텍스트: 듀오링고 스타일의 활기찬 UI 구현.

태스크: Tailwind CSS를 사용하여 전역 디자인 토큰(그린/블루 색상)을 설정하고, 모바일 중심의 네비게이션 바와 레이아웃 컴포넌트를 작성하세요.

[M2] 핵심 기능 개발 (FEAT-1: 교정 엔진)
컨텍스트: Gemini API를 활용한 '의도 유지' 교정 로직.

태스크: 사용자가 입력한 텍스트를 Gemini Pro API에 전달하는 서버 액션을 작성하세요. 프롬프트에는 "사용자의 의도를 유지하되 세련된 문어체와 실전 구어체 두 버전을 생성하고, 다정한 친구 말투로 피드백하라"는 지시가 포함되어야 합니다.

[M3] 데이터 영속성 및 결과 화면
컨텍스트: 일기 저장 및 결과 UI.

태스크: 교정 결과를 Supabase에 저장하고, 사용자가 '교정 전/후'를 명확히 비교하며 학습할 수 있는 카드 형태의 UI를 구현하세요.

7. Coding Convention & AI Collaboration Guide
신뢰하되 검증하라: AI가 생성한 코드에 Gemini API 키나 Supabase 비밀번호가 노출되지 않도록 .env 관리를 철저히 합니다.

모듈화: UI 컴포넌트(Atomic)와 비즈니스 로직(Server Actions)을 엄격히 분리하여 1인 운영 시 유지보수가 쉽도록 합니다.

AI 소통 원칙: "이 코드를 더 간결하게 만들어줘" 보다는 "성능 최적화와 가독성 관점에서 이 함수를 리팩토링해줘"라고 구체적으로 지시합니다.
